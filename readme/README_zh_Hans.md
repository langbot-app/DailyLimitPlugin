# 每日对话限制插件

按用户限制每天的对话次数。

## 功能

- **按用户限制**：为每个用户设置每天的最大对话次数
- **自定义提示语**：配置超出限制时显示的提示消息
- **静默模式**：可选择在达到限制时静默丢弃消息，不做任何回复
- **时区感知重置**：支持配置重置时区和时间点，默认为北京时间 0 点
- **自动清理**：旧的使用数据会自动清理，避免存储膨胀

## 配置项

| 选项 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `daily_limit` | 整数 | `50` | 每个用户每天的最大对话次数。设为 `0` 则不限制。 |
| `limit_message` | 字符串 | `您今天的对话次数已达上限，请明天再来吧~` | 超出限制时显示的提示语。 |
| `reset_timezone_offset` | 整数 | `8` | 重置时区 UTC 偏移（小时）。如 `8` 为东八区（中国），`9` 为东九区（日本）。 |
| `reset_hour` | 整数 | `0` | 每天重置计数器的时间点（按配置的时区）。默认为 0 点。 |
| `silent_mode` | 布尔值 | `false` | 开启后，超出限制的消息将被静默丢弃。 |

## 工作原理

插件监听私聊和群聊的普通消息事件。每条消息到达时，插件会检查发送者当天的使用次数是否达到上限：

- **静默模式关闭**：回复配置的提示语，并阻止后续处理。
- **静默模式开启**：静默阻止后续处理，不做任何回复。

使用次数在配置的时区和时间点自动重置（默认为北京时间每天 0 点）。
